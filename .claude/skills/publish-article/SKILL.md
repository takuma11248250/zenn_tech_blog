# Zenn記事を作成してコミット・プッシュする

ユーザーが投げたテキストをもとに記事を生成し、mainから新しいブランチを切ってコミット・プッシュします。

## 受け取るもの

$ARGUMENTS にユーザーが書いた記事のアイデア・内容のテキストが入ります。

## 手順

以下を順番に実行してください。

### 1. 既存記事を参考にする

`articles/` ディレクトリ内の既存記事を1〜2本読んで、文体・構成・フロントマターの形式を把握する。

### 2. 記事を生成する

以下のルールに従って記事を生成する。

- **文体**：ですます調（〜です、〜ます）
- **フロントマター**：既存記事と同じ形式。`published: true`
- **注意セクション**：所属会社の公式見解ではない旨を冒頭に記載
- **構成**：$ARGUMENTS の内容を整理して、読みやすいセクション構成にまとめる
- **まとめ**：最後にまとめセクションを追加する
- **絵文字**：本文中に絵文字は使わない
- **強調**：本文を書き終えたら全体を見直し、重要な主張・結論部分は `**太字**` で強調する

### 3. ファイルを保存する

ファイル名はランダムな14文字の16進数文字列にする。

```bash
SLUG=$(openssl rand -hex 7)
```

`articles/${SLUG}.md` に記事を書き出す。

### 4. タイトルを確認する

生成した記事のフロントマターから `title:` の値を取得しておく。

### 5. mainを最新化してブランチを切る

```bash
git checkout main
git pull origin main
git checkout -b add-${SLUG}
```

### 6. コミットする

コミットメッセージは記事の `title:` の値をそのまま使う。

```bash
git add articles/${SLUG}.md
git commit -m "<記事のタイトル>"
```

### 7. プッシュする

```bash
git push -u origin add-${SLUG}
```

## 完了後

プッシュ完了後、GitHubのPR作成URLをユーザーに伝える。
